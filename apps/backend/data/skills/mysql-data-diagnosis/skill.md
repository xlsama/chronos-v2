# MySQL 数据诊断方法论

## 适用场景

- 数据异常（价格为零、数量为负）
- 数据不一致（外键关系断裂）
- 批量操作导致的数据污染
- 订单金额异常
- 数据库配置错误

## 诊断步骤

### 1. 确认异常范围

- 根据告警信息确定涉及的表和字段
- 使用 SELECT 查询统计异常记录数量
- 通过 GROUP BY 分析异常数据的分布特征（按分类、时间段等维度）

### 2. 查看错误日志

- 查询 app_errors 表获取相关错误日志
- 按时间排序，找到问题首次出现的时间点
- 分析日志中的错误模式和关联服务

### 3. 关联分析

- 通过 JOIN 查询关联表，追溯异常数据的来源
- 检查外键关系是否完整
- 对比正常数据和异常数据的差异

### 4. 定位根因

- 结合异常数据的分布特征和时间线
- 判断是人为操作错误（批量 SQL）还是系统 bug
- 确认影响范围和业务影响程度

### 5. 输出诊断结论

- 总结根因
- 列出受影响的数据范围
- 提供修复建议（如需要数据回滚或修正）

## 常见模式

| 症状 | 根因 | 查询 |
|------|------|------|
| 价格为零 | 批量更新错误 | `SELECT * FROM products WHERE price = 0` |
| 订单金额异常 | 价格数据污染 | `SELECT * FROM orders WHERE total <= 0` |
| 外键断裂 | 数据删除不完整 | 使用 LEFT JOIN 检查 |

## 安全注意事项

- 只执行 SELECT 查询，不执行修改操作
- 大表查询务必加 LIMIT
