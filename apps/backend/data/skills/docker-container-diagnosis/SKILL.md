# Docker 容器诊断方法论

## 适用场景

- 容器频繁重启或退出
- 容器资源使用异常（CPU/内存）
- 容器网络不通
- 镜像拉取失败
- 容器日志分析

## 诊断步骤

### 1. 列出容器状态

- 列出所有运行中和已停止的容器
- 关注 STATUS 列：Exited（退出码）、Restarting
- 检查 CREATED 和 STATUS 时间，判断容器存活时长

### 2. 检查容器日志

- 获取异常容器的最近日志
- 查找错误、异常、panic 等关键信息
- 关注容器启动阶段的日志（配置加载、端口绑定等）

### 3. 检查容器详情

- 查看容器的配置信息（环境变量、端口映射、卷挂载）
- 检查资源限制设置（memory limit、cpu quota）
- 查看重启策略和健康检查配置

### 4. 检查资源使用

- 查看容器的实时资源使用（CPU/内存/网络/磁盘IO）
- 对比资源限制，判断是否接近上限
- 检查是否有资源竞争

### 5. 检查网络

- 查看容器的网络配置和连接的网络
- 检查端口映射是否正确
- 验证容器间的网络连通性

### 6. 输出诊断结论

- 总结容器异常的根因
- 列出受影响的服务
- 提供修复建议

## 常见模式

| 退出码 | 含义 | 诊断方向 |
|--------|------|----------|
| 0 | 正常退出 | 检查是否期望持续运行 |
| 1 | 应用错误 | 查看容器日志 |
| 137 | OOMKilled (SIGKILL) | 增加内存限制 |
| 139 | 段错误 (SIGSEGV) | 检查应用 bug |
| 143 | SIGTERM | 外部信号终止 |

## 安全注意事项

- 只读操作，不执行 start/stop/rm/exec 等修改操作
- 环境变量中可能包含敏感凭据
