# API 网关限流架构文档

## 概述

API Gateway 是平台统一流量入口，负责请求路由、身份验证、限流与网关层异常记录。运行时依赖 Redis 存储配置与部分网关状态。

## 服务链路

- 上游：客户端、发布脚本、alert-manager
- 核心组件：api-gateway
- 运行时存储：Redis
- 下游影响：`/api/orders`、`/api/products`、`/api/users`、`/api/health`

当多个业务 endpoint 同时出现 429，而健康检查接口仍然可用时，通常说明问题更接近网关的限流相关运行态，而不是整个网关完全不可用。

## Redis 键空间约定

### 限流配置

- 模式：`ratelimit:config:{endpoint}`
- 值类型：JSON
- 用途：描述 endpoint 的限流窗口和阈值

### 限流计数器

- 模式：`ratelimit:counter:{endpoint}:{client}`
- 值类型：整数
- 用途：记录时间窗口内的请求计数

### 功能开关

- 模式：`feature:{name}`
- 值类型：字符串或轻量配置
- 用途：控制维护模式、网关策略等运行时行为

### 网关运行日志

- 模式：`errorlog:{id}`
- 值类型：JSON
- 用途：记录网关侧错误、异常限流和告警关联信息

## 运行约束

- `/api/health` 通常作为低风险健康检查接口，和核心业务接口的限流策略可以不同
- 业务接口的限流表现会同时受到配置值、当前计数器状态和全局功能开关影响
- 同一时间多个 endpoint 异常，常见于共享配置、批量变更或网关层统一策略异常

## 观测建议

- 先区分“流量真实过高”和“网关运行态异常”两类情况
- 观察异常是否集中在某些 endpoint，还是覆盖多个业务路径
- 结合 Redis 中的配置、计数器和网关运行日志判断异常发生在哪一层
